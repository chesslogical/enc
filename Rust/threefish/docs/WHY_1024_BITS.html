2. Why the bit‑count misleads

  • Asymmetric keys (RSA, DH) rely on special mathematical problems.
    Best attacks grow sub‑exponentially, so key sizes must be much larger (2048 / 3072 / 4096 bit) for comfort.
  • Symmetric keys (AES, Threefish) resist attack only by exhaustive search.
    Attack cost doubles with every extra bit; 128 bits is already considered safe for decades.

3. Implications for the threefish tool

  1. Key length – the program’s 128‑byte (1024‑bit) secret key is far beyond practical brute force.
  2. Main risks – key disclosure, nonce reuse, and implementation bugs matter more than key size.
  3. Compliance – no policy forbids 1024‑bit symmetric keys; they comfortably exceed AES‑128 strength.

5. Bottom line

RSA‑1024 is obsolete because factoring advances have outpaced its security margin.
Threefish‑1024 offers a security margin vastly beyond practical need; focus on
key management, nonce discipline, and modern asymmetric wrappers instead of enlarging an already enormous symmetric key.
