🔐 SecureFile

A production-ready file encryption CLI using AES-256-GCM-SIV in Rust.

This tool provides in-place encryption and decryption of files using AES-256-GCM-SIV, offering robust security with nonce-misuse resistance, authentication, and memory hygiene.

⚙ Features

  • ✅ AES-256-GCM-SIV (RFC 8452) with random nonce
  • ✅ In-place file encryption (overwrites original)
  • ✅ Key wiping from memory after use
  • ✅ Strict 32-byte key file requirement
  • ✅ Built with security-focused crates: aes-gcm-siv, zeroize, anyhow

🚀 Usage

1. Install Rust

https://rustup.rs

2. Clone and Build

git clone https://github.com/yourusername/securefile
cd securefile
cargo build --release

3. Generate a Key File

# Linux/macOS
head -c 32 /dev/urandom > key.key

# Windows (PowerShell)
[IO.File]::WriteAllBytes("key.key", (1..32 | ForEach-Object { Get-Random -Minimum 0 -Maximum 256 }))

4. Encrypt a File

./target/release/securefile encrypt secret.txt

5. Decrypt a File

./target/release/securefile decrypt secret.txt

🔒 Security Notes

  • AES-GCM-SIV is resistant to nonce reuse, but random nonces are still used per file for best practice.
  • The key file must be securely stored and backed up — there is no recovery if lost.
  • This tool does not (yet) support password-based key derivation — see roadmap below.
